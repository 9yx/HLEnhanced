114 // ------------------------------------------------------------------------------------------------ //
115 // Displacement neighbor rules
116 // ------------------------------------------------------------------------------------------------ //
117 //
118 // Each displacement is considered to be in its own space:
119 //
120 // NEIGHBOREDGE_TOP
121 //
122 // 1 --- 2
123 // | |
124 // NEIGHBOREDGE_LEFT | | NEIGHBOREDGE_RIGHT
125 // | |
126 // 0 --- 3
127 //
128 // NEIGHBOREDGE_BOTTOM
129 //
130 //
131 // Edge edge of a displacement can have up to two neighbors. If it only has one neighbor
132 // and the neighbor fills the edge, then SubNeighbor 0 uses CORNER_TO_CORNER (and SubNeighbor 1
133 // is undefined).
134 //
135 // CORNER_TO_MIDPOINT means that it spans [bottom edge,midpoint] or [left edge,midpoint] depending
136 // on which edge you're on.
137 //
138 // MIDPOINT_TO_CORNER means that it spans [midpoint,top edge] or [midpoint,right edge] depending
139 // on which edge you're on.
140 //
141 // Here's an illustration (where C2M=CORNER_TO_MIDPOINT and M2C=MIDPOINT_TO_CORNER
142 //
143 //
144 // C2M M2C
145 //
146 // 1 --------------> x --------------> 2
147 //
148 // ^ ^
149 // | |
150 // | |
151 // M2C | | M2C
152 // | |
153 // | |
154 //
155 // x x x
156 //
157 // ^ ^
158 // | |
159 // | |
160 // C2M | | C2M
161 // | |
162 // | |
163 //
164 // 0 --------------> x --------------> 3
165 //
166 // C2M M2C
167 //
168 //
169 // The CHILDNODE_ defines can be used to refer to a node's child nodes (this is for when you're
170 // recursing into the node tree inside a displacement):
171 //
172 // ---------
173 // | | |
174 // | 1 | 0 |
175 // | | |
176 // |---x---|
177 // | | |
178 // | 2 | 3 |
179 // | | |
180 // ---------
181 //
182 // ------------------------------------------------------------------------------------------------ //
