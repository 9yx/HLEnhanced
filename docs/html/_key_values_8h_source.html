34 // Purpose: Simple recursive data access class
35 // Used in vgui for message parameters and resource files
36 // Destructor deletes all child KeyValues nodes
37 // Data is stored in key (string names) - (string/int/float)value pairs called nodes.
38 //
39 // About KeyValues Text File Format:
40 
41 // It has 3 control characters '{', '}' and '"'. Names and values may be quoted or
42 // not. The quote '"' charater must not be used within name or values, only for
43 // quoting whole tokens. You may use escape sequences wile parsing and add within a
44 // quoted token a \" to add quotes within your name or token. When using Escape
45 // Sequence the parser must now that by setting KeyValues::UsesEscapeSequences( true ),
46 // which it's off by default. Non-quoted tokens ends with a whitespace, '{', '}' and '"'.
47 // So you may use '{' and '}' within quoted tokens, but not for non-quoted tokens.
48 // An open bracket '{' after a key name indicates a list of subkeys which is finished
49 // with a closing bracket '}'. Subkeys use the same definitions recursively.
50 // Whitespaces are space, return, newline and tabulator. Allowed Escape sequences
51 // are \n, \t, \\, \n and \". The number character '#' is used for macro purposes
52 // (eg #include), don't use it as first charater in key names.
