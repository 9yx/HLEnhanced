29 // Purpose: A menu is a list of items that can be selected with one click, navigated
30 // with arrow keys and/or hot keys, and have a lit behavior when mouse over.
31 // It is NOT the button which opens the menu, but only the menu itself.
32 //
33 // Behaviour spec:
34 // Menu navigation can be done in 2 modes, via keyboard keys and via mouse.
35 // Clicking on menu button opens menu.
36 // Only one item in a menu is highlighted at a time.
37 // Only one submenu in a menu is open at a time.
38 // Disabled menuitems get highlighted via mouse and keys but will not activate.
39 //
40 // Mouse:
41 // Moving mouse into a menuitem highlights it.
42 // If the menuitem has a cascading menu, the menu opens when the mouse enters
43 // the menuitem. The cascading menuitem stays highlighted while its menu is open.
44 // No submenu items are highlighted by default.
45 // Moving the mouse into another menuitem closes any previously open submenus in the list.
46 // Clicking once in the menu item activates the menu item and closes all menus.
47 // Moving the mouse off a menuitem unhighlights it.
48 // The scroll bar arrows can be used to move up/down the menu one item at a time.
49 // The clicking and dragging on the scroll bar nob also scrolls the menu items.
50 // If a highlighed menuitem scrolls off, and the user then begins navigating via keys,
51 // the menu will snap the scroll bar so the highlighted item is visible.
52 // If user has been navigating via keys, moving the mouse over a menu item
53 // highlights it.
54 // Mousewheel:
55 // You must have the mouse inside the menu/scroll bar to use the wheel.
56 // The mouse wheel moves the highlighted menuitem up or down the list.
57 // If the list has no scroll bar the wheel will cycle from the bottom of the list
58 // to the top of the list and vice versa.
59 // If the list has a scrollbar the mouse wheel will stop at the top or bottom
60 // of the list.
61 // If the mouse is over the scroll bar no items are highlighted.
62 // Keyboard:
63 // When a menu is opened, no items are highlighted.
64 // If a menuitem has a cascading menu it does not open when the item is highlighted.
65 // The down arrow selects the next item in the list.
66 // (first item if none are highlighted and there is a scrollbar).
67 // The up arrow selects the previous item in the list
68 // (first item if none are highlighted and there is a scrollbar, last item if none are
69 // highlighted and there is no scrollbar).
70 // Selecting a new menuitem closes any previously open submenus in the list.
71 // The enter key activates the selected item and closes all menus.
72 // If the selected item has a cascading menu, activating it opens its submenu.
73 // These may also be activated by pressing the right arrow.
74 // Pressing the left arrow closes the submenu.
75 // When the submenu is opened the cascading menuitem stays highlighted.
76 // No items in the submenu are highlighted when it is opened.
77 //
78 // Note: Cascading menuitems in menus with a scrollbar is not supported.
79 // Its a clunky UI and if we want this we should design a better solution,
80 // perhaps along the lines of how explorer's bookmarks does it.
81 // It currently functions, but there are some arm/disarm bugs.
82 //
83 //
84 //-----------------------------------------------------------------------------
