3 // .xzp creation and manipulation.
4 //
5 // XZip file format:
6 // [xZipHeader]
7 //
8 //
9 // Regular directory:
10 // [xZipDirectoryEntry 1]
11 // [xZipDirectoryEntry 2]
12 // [xZipDirectoryEntry 3]
13 // [...]
14 //
15 // Preload offset points here:
16 // Preload directory:
17 // [xZipDirectoryEntry 1]
18 // [xZipDirectoryEntry 2]
19 // [xZipDirectoryEntry 3]
20 // [...]
21 // [unsigned short directory to preload mapping 1]
22 // [unsigned short directory to preload mapping 2]
23 // [unsigned short directory to preload mapping 3]
24 // [...]
25 // [regular preload data]
26 //
27 // [consolidated file data]
28 //
29 // (v2+ only):
30 // The filename string table. This is stored at the end of the file because in most cases it will rarely ever be read:
31 // [xZipFilenameEntry 1]
32 // [xZipFilenameEntry 2]
33 // [xZipFilenameEntry 3]
34 // [...]
35 // [filename string data]
36 //
37 // The filename strings are currently only used for debugging purposes, though
38 // code could be written to use them for findfirst / findnext functionality.
39 //
40 //===========================================================================
41 //
42 // .xzp format version history:
43 //
44 // 1: Initial version
45 // 2: Filename strings added, still reads version 1.
46 // 3: File dates added to the filename strings structure.
47 // 4: No more filename length - filenames are now offsets
48 //
49 //===========================================================================
